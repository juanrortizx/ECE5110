\begin{abstract}
This article models the behavior of a temperature sensor in an industrial setting using an ordinary differential equation (ODE), solved numerically with the Runge-Kutta method in Python.
\end{abstract}

\section{Objective}
\label{sec:intro}
The objective is to solve a temperature sensor ODE using the Runge-Kutta method.

\section{Method}
We can model the temperature sensor's behavior using a first-order ODE of the form:
\[\hspace{0.4in}
\frac{dT}{dt} = -a(T - T_{ambient}) + K_{p}(T_{setpoint} - T)
\]

\begin{center}
\begin{minipage}{0.75\linewidth} % adjust width as you like
\begin{minted}[fontsize=\small, frame=single]{python}
def f(t, y):
    return -a * (y - T_ambient) 
                  + Kp * (T_set - y)
\end{minted}
\end{minipage}
\end{center}

\noindent Where:

\begin{itemize}
    \item $T$: sensor temperature
    \item $t$: time
    \item $a$: positive cooling rate constant
    \item $T_{ambient}$: ambient temperature
    \item $K_{p}$: proportional gain constant
    \item $T_{setpoint}$: desired temperature
\end{itemize}

In this model, the term $-a(T - T_{ambient})$ represents cooling toward the ambient temperature, while $K_{p}(T_{setpoint} - T)$ is a proportional control term driving the temperature toward the desired setpoint.
\vspace{0.1in}

\noindent Although this ODE has an analytical solution, numerical methods are preferred in practice due to noise and system complexity. Here, we use the fourth-order Runge–Kutta (RK4) method.
\vspace{0.1in}

\noindent RK4 advances the solution using: 
\[\hspace{0.4in}
k_1 = f(t_n, y_n)
\]
\[\hspace{0.4in}
k_2 = f(t_n + \frac{h}{2}, y_n + \frac{h}{2}k_1)
\]
\[\hspace{0.4in}
k_3 = f(t_n + \frac{h}{2}, y_n + \frac{h}{2}k_2)
\]
\[\hspace{0.4in}
k_4 = f(t_n + h, y_n + hk_3)
\]
\[\hspace{0.4in}
y_{n+1} = y_n + \frac{h}{6}(k_1 + 2k_2 + 2k_3 + k_4)
\]

\noindent In python this can be implemented as follows:
\begin{center}
\begin{minipage}{0.75\linewidth} % adjust width as you like
\begin{minted}[fontsize=\small, frame=single]{python}
for i in range(1, len(steps)):
  x_prev = steps[i - 1]
  y_prev = y[i - 1]
  k1 = f(x_prev, y_prev)
  k2 = f(x_prev + h / 2, 
         y_prev + h * k1 / 2)
  k3 = f(x_prev + h / 2, 
         y_prev + h * k2 / 2)
  k4 = f(x_prev + h, 
         y_prev + h * k3)
  y[i] = y_prev + h * (k1 + 2 * k2 + 
                            2 * k3 + 
                                k4) / 6
\end{minted}
\end{minipage}
\end{center}

\noindent This code iteratively computes the slopes $k_1$, $k_2$, $k_3$, and $k_4$ at each time step and updates the solution $y$ accordingly.
\vspace{0.1in}

\noindent We will simulate noise in the system to closer match real-world conditions. This can be done by adding a random noise term to the function. For example:

\begin{center}
\begin{minipage}{0.75\linewidth} % adjust width as you like
\begin{minted}[fontsize=\small, frame=single]{python}
noise = np.random.normal(0.0, 
                         noise_std, 
                         size=len(y)
                         )
y_noisy = y + noise
\end{minted}
\end{minipage}
\end{center}
